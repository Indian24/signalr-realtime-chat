You are a senior .NET architect and Developer Relations Engineer.

Your task is to build a lightweight but production-quality sample application demonstrating real-time communication using ASP.NET Core and SignalR.

The goal is NOT just to build a chat app, but to create a developer-focused reference project that teaches developers how to use SignalR correctly.

This project will be used as a sample application similar to those created by Developer Relations Engineers at software companies like GrapeCity.

Project Requirements:

Technology Stack:

* ASP.NET Core (.NET 8)
* C#
* ASP.NET MVC
* SignalR
* JavaScript (client side)
* Minimal dependencies
* SQLite or in-memory storage (avoid heavy databases)
* No AI/ML or Python components

Architecture Requirements:
Follow a clean and educational architecture:

/Controllers
Handles HTTP requests and MVC routing

/Hubs
Contains SignalR hubs responsible for real-time communication

/Models
Domain models such as ChatMessage and User

/Services
Business logic for message handling

/Data
Database context (lightweight)

Views/
MVC views for UI

wwwroot/
Client-side scripts and CSS

Functional Features:

1. Real-time messaging using SignalR
2. Broadcast messages to all connected users
3. Display chat messages instantly without refreshing
4. Maintain a simple message history
5. Show connected users
6. Lightweight UI for demo purposes

Developer Experience Goals:

* Code must be readable and heavily commented
* Avoid unnecessary abstraction
* Focus on clarity and learning value
* Separate HTTP logic from real-time logic
* Demonstrate best practices for SignalR hubs

Program.cs Requirements:

* Configure MVC
* Register SignalR services
* Map the chat hub endpoint

Example:
builder.Services.AddSignalR();
app.MapHub<ChatHub>("/chatHub");

SignalR Hub Requirements:
Create ChatHub with:

SendMessage(user, message)

This method should broadcast messages to all connected clients.

Client Side Requirements:
Use JavaScript SignalR client library to:

* Connect to the hub
* Send messages
* Receive broadcast messages
* Update UI instantly

Documentation Requirements:

Create the following documentation files:

README.md
Explain the purpose of the project, architecture, and learning outcomes.

ARCHITECTURE.md
Explain how real-time messaging works and how SignalR hubs interact with MVC controllers.

docs/signalr-basics.md
Explain SignalR concepts in beginner-friendly language.

Performance Requirements:
Keep the project lightweight so it runs smoothly in environments like Replit.

Avoid:

* Docker
* Large frameworks
* Complex front-end frameworks

Output Requirements:
Return the FULL project structure with all files and code.

Also explain:

* How to run the project
* Which URL opens the chat page
* How developers can extend the system
